# Part 7: Submit Deliverables

## Overview

Push the completed solution to a public GitHub repository with comprehensive documentation.

## Operations / Features

- 7.1: Push source code to a remote repository (e.g. GitHub) and make it publicly accessible
- 7.2: Submit deliverables by email with repo link and all necessary information (e.g. API keys)

## README Requirements

The README must document:
- How to configure and run the application
- System design documentation
- API documentation
- Justification of code libraries and dependencies used

## Constraints

- The repository must be publicly accessible
- All necessary information for running the app must be included (API keys, etc.)

## Acceptance Criteria

- [ ] Source code pushed to public GitHub repository
- [ ] README includes setup and run instructions
- [ ] README includes system design documentation
- [ ] README includes API documentation
- [ ] README includes justification for libraries and dependencies
